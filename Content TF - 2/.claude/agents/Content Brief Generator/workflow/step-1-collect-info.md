# Step 1: Collect Required Information

Before generating headlines, you MUST verify you have all required information. If any required field is missing, ask the user to provide the missing information.

---

## Required Fields

- **Primary Keyword** — The main keyword/phrase this page targets
- **Secondary Keywords** — Additional keywords (comma-separated list)
- **Client Name** — Name of the client/brand (default: **Visual Planning**)
- **Page Name** — Internal name for this page
- **Page Type** — Type of page being created (e.g., landing page, blog, pillar page, service page, category page)
- **ICP Information** — Ideal Customer Profile details for this page: target role, industry, pain points, and goals. See the Visual Planning ICP reference below.
- **Competitor Domains** — Competitor domain(s) to analyze (comma-separated if multiple)

---

## Visual Planning ICP Reference

When the user has not explicitly provided ICP Information, read the following resource files to populate the field:

- **[../Resources/VP Ideal Customer Profile.docx](../Resources/VP%20Ideal%20Customer%20Profile.docx)** — Target industries, personas, pain points, and goals
- **[../Resources/Completed Client Questionnaire.docx](../Resources/Completed%20Client%20Questionnaire.docx)** — Client-provided context on brand, product, target market, and competitors

Use these files to extract: target roles, industries, core pain points, goals, key value propositions, and competitor domains.

---

## Optional Fields

- **Existing Page URL** — URL if updating an existing page (helps understand current approach)
- **Page Context / Additional Page Info** — Any extra details about the page, its purpose, or specific requirements (use to align output with expectations)

---

## How to Collect Missing Information

1. Review the user's initial message carefully for any provided information
2. If ANY required field is missing, ask the user to provide the specific missing fields
3. Do NOT proceed to Step 2 until all required fields are collected
4. Frame questions clearly and professionally

---

## Example Collection

```
Collected Information:
- Primary Keyword: resource scheduling software
- Secondary Keywords: staff scheduling software, resource planning tool, team scheduling
- Client Name: Visual Planning
- Page Name: Resource Scheduling Software
- Page Type: Landing page
- ICP Information: Operations Managers and Project Managers in manufacturing, field service, and consulting. Pain points: scheduling complexity, spreadsheet limitations, team errors, manual admin. Goals: visual scheduling, real-time collaboration, smarter resource assignment.
- Domains: whenIwork.com, smartsheet.com, shiftplanning.com
- Existing Page URL: (none provided)
- Page Context / Additional Page Info: (none provided)
```
